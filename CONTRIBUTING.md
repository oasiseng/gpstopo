# Contribution Guidelines

We welcome contributions from mapping professionals and open-source developers.

## How to Contribute

1. Fork the repository and create a feature branch for your updates.
2. Use Markdown for documentation and keep files lightweight and accessible.
3. Provide clear commit messages summarizing the intent of each change.
4. Submit a pull request describing the improvements and reference any relevant
   issues or workflows.

## Best Practices

- Tag all technical updates with the corresponding project, workflow, or
  template.
- Follow GPSTopo naming conventions and version control standards.
- Do not upload proprietary, client-specific, or sensitive operational data.
- Include examples or screenshots when updating templates or visual assets.

## Code of Conduct

By participating, you agree to uphold the [Code of Conduct](docs/code-of-conduct.md).

Thank you for helping us improve the GPSTopo Codex.
